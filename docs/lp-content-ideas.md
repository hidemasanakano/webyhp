# LP Content Ideas — 採用サイト網羅性ブレスト

**対象**: 株式会社WEBY 新卒採用LP（2027年卒向け）
**作成**: 2026-04-22（v1.0）
**位置づけ**: `design.md` がデザインシステム（現状の実装事実）を扱うのに対し、本書は**まだ実装されていない改善アイデアのバックログ**。人事・ブランド決裁を経て Issue 化・実装するための素材集。

---

## 0. 使い方

- 採用戦略会議・人事ミーティングの議題の叩き台として使う
- 決裁が取れたアイデアは GitHub Issue に転記して実装タスクに昇格
- 未決裁のアイデアはここで寝かせる（メモ帳として）
- 新しいアイデアは随時追記。却下されたものは **却下理由を併記**して残す（同じ議論の再発を防ぐ）

### 提案 3: 選考フロー ビジュアルタイムライン 🔴 P0

**意図**: 「応募してからどうなるか」の不透明感が離脱要因。CRO でいう Effort objection への直接回答

**配置**: 募集要項（Recruit）の**直下**、既存の Post-Recruit CTA の**前**

**Visual コンセプト:**

```
ENTRY   →   1次選考   →   面接 (最大3回)   →   内定
  ↓          ↓             ↓                  ↓
 3分       約1週間       計3週間         約2ヶ月で完了

[各ステップに:]
- 所要日数
- 何が問われるか（例: 1次=自己理解、2次=業務適性、3次=代表面接）
- 服装目安（スーツ or ビジカジ）
- オンライン/対面
```

**必要情報**: 人事より「実際の選考フロー・所要期間」確認
**実装**: タイムラインコンポーネント（Training Flow と同じパターン流用可）

**意図**: 「今すぐ応募」だけが導線では、検討中層を取りこぼす。複数の低コミットメント入口で囲い込む

**配置**: Recruit セクションの**前**、または Post-Recruit CTA のすぐ後

┌────────────────┬────────────────┬────────────────┐
│  SUMMER IS     │  YEAR-ROUND IS │  INFO SESSION  │
│  28サマー       │  通年長期      │  説明会27       │
│  インターン    │  インターン    │  (カジュアル面談) │
│                │                │                │
│  [夏休み集中]  │  [週2日〜OK]   │  [毎週水曜 19:00]│
│  5日間プログラム│  最大12ヶ月    │  30分オンライン │
│                │                │                │
│  [APPLY →]     │  [APPLY →]     │  [RESERVE →]   │
│                │                │                │
│  次回: 2028/07  │  通年募集中    │  次回: 2026/05/07│
└────────────────┴────────────────┴────────────────┘

**必要情報**:
- サマーインターン URL + 日程 + 定員
- 長期インターン URL + 条件（時給・週数）
- 説明会予約システム（Google Form or TimeRex 等）+ 日程

### 提案 8: モバイル下固定 CTA バー 🔴 P0

**意図**: モバイル閲覧率 65-75%。スクロール中いつでもタップできる応募入口

**配置**: モバイルのみ、画面下に常時 70px の固定バー

**Visual コンセプト:**

```
┌─────────────────────────────────────┐
│  [コンテンツ領域]                    │
│                                     │
│  ...                                │
│                                     │
├─────────────────────────────────────┤
│ [📅 説明会予約] [🔴 ENTRY  約3分]  │ ← 下固定 70px
└─────────────────────────────────────┘
```

**非表示にすべきタイミング**: Footer が画面に入った時点で消す（既にフッター CTA に到達しているため）
**実装難易度**: 低（CSS fixed + IntersectionObserver）

### 提案 10: 親世代向け FAQ 専用ブロック 🟢 P2

**意図**: 内定承諾時の「親ブロック」を事前に解消。内定獲得 → 承諾率向上

**配置**: FAQ 内で「ご家族の方へ」タブ分割、または FAQ の下に別ブロック

**含めるべき FAQ 例:**
- 会社の従業員数・売上規模は？
- 社会保険は何組合？
- 離職率は？
- 育休・産休の実績は？
- 昇進事例は？
- 親が会社見学できる機会はある？
